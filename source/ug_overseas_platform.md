# UG Overseas Platform - 海外增長業務平台

## 專案概述

UG Overseas Platform 是一個大型企業級 **Monorepo** 前端專案，服務於字節跳動海外增長業務線，涵蓋媒體資產管理、財務結算、供應商採購、數據看板等多個業務領域。專案採用統一的技術架構和工程規範，支持多個業務子系統的協同開發與獨立部署。

## 業務模組 (Foundation)

### 1. 供應商採購管理系統 (Recruit)
全流程供應商招募與採購管理平台，支援：
- **需求管理**：建立、編輯、審核採購需求
- **評審管理**：供應商資質評審與審批流程
- **採購任務**：任務分派、進度追蹤
- **價格策略**：批量上傳、動態定價配置
- **供應商管理**：供應商資訊維護與績效評估

### 2. 媒體資產管理平台 (Media)
廣告投放與媒體資源管理中心，功能包括：
- **消耗錄入**：批量/單筆廣告消耗數據錄入
- **帳戶管理**：管家帳戶、廣告帳戶配置
- **監控預警**：預算監控、ROI 預警、監控歷史
- **返點管理**：媒體返點配置與結算
- **OneLink 歸因**：歸因連結建立與效果追蹤
- **目標管理**：投放目標設定與達成追蹤

### 3. 數據看板 (Lingsee)
實時業務數據視覺化平台：
- **實時 DAU**：日活數據即時監控
- **產品概覽**：核心業務指標儀表板

### 4. 活動推廣管理 (Promotion)
行銷活動配置與管理系統

### 5. 財務結算系統 (UG Financial)
海外業務財務對帳與結算管理

### 6. 通用元件庫 (Common)
跨模組共享元件與工具函數：
- `EmployeeSelect` - 員工選擇器
- `UploadFile` - 檔案上傳元件
- `ModalFormRender` - 彈窗表單渲染器
- `LarkAvatar` - 飛書頭像元件
- 其他通用 Hooks、Utils、Services

## 技術亮點

### 1. Monorepo 架構設計
採用 **Rush + PNPM Workspaces** 管理 50+ 子專案，實現：
- 統一依賴管理，避免版本衝突
- 跨專案程式碼共享與模組複用
- 獨立部署與建構隔離

### 2. 複雜表單互動元件開發
基於 JSON Schema 的動態表單解決方案 (`@byted/ug-form-render`)：
- 開發大量業務表單互動元件，涵蓋級聯選擇、動態欄位、條件顯示等複雜場景
- 配置驅動的表單渲染，支援複雜校驗規則與欄位聯動邏輯
- 表單模板化管理，提升開發效率與一致性

### 3. 高精度數值計算處理
針對財務、報價等業務場景的浮點數精度問題：
- 實作浮點數精度計算方案，避免 JavaScript 浮點數運算誤差
- 自定義 Yup 校驗方法 `maxDecimals`，確保小數位數符合業務規範
- 支援批量數值（逗號/分號分隔）的精度校驗

### 4. 函數快取工具優化渲染效能
針對大量數據渲染場景，開發通用函數快取工具：
- **`cache`**：單一結果快取，適用於參數穩定的計算場景
- **`cacheMany`**：多參數組合快取（LRU 策略），支援動態參數的重複計算優化
- 採用 `fast-json-stable-stringify` 序列化參數，確保快取鍵一致性
- 應用於 Schema 生成、選項計算等高頻場景，顯著減少重複計算時間

### 5. Web Worker 多線程 CSV 校驗
針對大型 CSV 表格上傳場景，實現非同步校驗方案：
- 利用 **Web Worker** 將校驗邏輯移至獨立線程，避免主線程阻塞
- 採用 **Comlink** 簡化 Worker 通訊，提供類 RPC 的調用體驗
- 基於 **Yup** 建構動態校驗 Schema，支援自定義校驗規則（如 `requiredIf`、`maxDecimals`）
- 校驗錯誤精確定位至 Excel 儲存格座標（如 `A3`、`B5`），提升使用者體驗

### 6. 微前端整合
透過 `@byted-armor/micro` 實現子應用動態載入：
- 按需載入業務模組
- 子應用獨立開發與部署
- 統一的路由與狀態管理

### 7. 完善的國際化方案
Starling 多語系解決方案：
- 自動化翻譯流程整合
- 語言包動態載入
- 支援 RTL 語系

### 8. 類型安全的 API 通訊
Ferry + Thrift IDL 生成強型別 Service 程式碼：
- 前後端協議一致性保障
- 自動化介面型別生成
- 減少執行時錯誤
